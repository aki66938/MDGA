# Plan12 - Agent Capability Hardening

项目代号：MDGA
文档定位：在 Plan11 完成 Agent Tool Runtime 底模（0.0.9）之后，本文定义「Agent 能力强化线」——把已经能跑通的工具循环，强化为真正能胜任多步开发任务、用户可控、可安全放权、可审阅的 Agent。本文承接 `Plan11-Agent-Tool-Runtime-And-Extensions.md`、`Plan05-Agent-Kernel.md`、`Plan06-Security-And-Permissions.md`。

---

## 1. 背景与定位

0.0.9 已经让 MDGA 具备 Agent 底模：模型自主调用工具、多轮推理、权限四档、审计可视化。但底模仍有明显短板，限制它胜任真实任务。

主创定调：

- **核心目标是 Agent 化**——通过对话让 DeepSeek 完成各种任务，已能做简单开发动作，下一步是强化 Agent 能力（高优先级）。
- **文件导入与问答排到最末**——必然涉及图像识别，而 DeepSeek API 当前不支持视觉，需额外嵌入识图模型才能补全，成本与依赖高，优先级最低，排在 MCP、Skills 之后。

因此整体优先级顺序确定为：

```text
Agent 能力强化（Plan12，本文）
  → MCP Adapter（Plan11 Phase 5）
  → Skills（Plan11 Phase 6）
  → 文件导入与问答（最末）
```

## 2. 第一性原则：一个强 Agent 取决于什么

从第一性原则拆解，Agent 实战能力由以下维度决定，逐项对照 0.0.9 现状：

| 维度 | 关键问题 | 0.0.9 现状 |
|------|----------|-----------|
| 能迭代多深 | 复杂任务需反复试错 | ⚠️ 轮数写死 5，太短；无智能终止上限 |
| 能否自纠 | 工具/命令失败后能否读错误换方案 | ❌ 失败结果回灌了，但无显式纠错引导 |
| 用户能否控场 | 黑盒跑多轮，中途想停停不了 | ❌ 无法中断；每轮叙述被丢弃，过程黑盒 |
| 能否安全放权 | 跑测试/git 必须开 Full Access | ⚠️ run_command 仅 Full Access，无低风险白名单；Ask Every Time 仍是假的 |
| 能否看懂项目 | 不知道有哪些文件就盲改 | ⚠️ 只能逐层 list_dir，无 repo map |
| 改动能否审阅 | 整文件覆盖，改完看不到 diff | ⚠️ 有 edit_file 但无 diff 展示/回滚底座 |

## 3. 强化线版本规划

按「对 Agent 实战能力的杠杆大小」排序，拆成 4 个可独立交付的 0.0.x 版本。

### 0.0.10 - Agent Loop 控制力（杠杆最大，先做）✅ 已实现并发版

目标：让 Agent「跑得深、看得见、停得下」。

任务：

- **智能终止 + 提高轮数上限**：保留「模型不再调用工具即结束」的自然终止；硬上限从 5 提高到 20（防失控兜底）。✅
- **多轮过程流式可见**：每轮开始时发出进度事件；模型在调用工具前产生的叙述/思考内容（intermediate content）不再丢弃，实时推送到前端，让用户看到「Agent 为什么这么做」。✅
- **可中断**：新增取消机制，用户能喊停正在运行的 Agent 任务；循环在每轮之间和每次工具执行后检查取消标志，安全收尾。✅

验收：

- 复杂多步任务（读→改→测→修）不再因 5 轮上限半途而废。✅
- 多轮过程中能实时看到工具动作与模型叙述，而非黑盒等待。✅
- 点击「停止」能中断正在运行的 Agent，已执行的工具结果保留。✅

### 0.0.11 - 工具执行内联可视化 ✅ 已实现并发版

> 插入版本：0.0.10 落地后实测发现，工具事件虽然实时推送，但前端把它们扎堆在底部独立面板，叙述与工具动作时间轴分离，用户仍有黑盒感。参考 Claude Code Desktop / Codex 的体验，工具调用应直接内联进对话流。该 UX 缺口优先级被提前，占用 0.0.11。

目标：消灭黑盒感，让「AI 思考到哪一步、调用了什么工具」在对话流中连贯可见。

任务：

- 前端消息模型从单一 `content: string` 重构为 `parts: MessagePart[]`（text 块 + tool 卡片交错）。✅
- 工具卡片**内联渲染**进对话流，与叙述文字按时间顺序交错。✅
- 工具卡片实时状态：running 显示动画点、succeeded 绿 ✓、failed 红 ✗ + 错误原因。✅
- 移除底部独立 activity panel。✅

验收：

- 多步任务中工具卡片交错出现在叙述之间，而非扎堆底部。✅
- 工具执行时能看到运行中→完成的实时状态变化。✅

### 0.0.12 - 安全放权 + 真实审批 ✅ 已实现并发版

目标：让 Agent 在不开 Full Access 的前提下能跑低风险命令，并补齐真实审批闭环。

任务：

- 修复权限裁决 Bug：AskEveryTime 此前对所有能力（含只读）都返回 AskUser，导致连读文件都报错、整模式不可用；改为只读直接放行，仅变更/命令/网络走审批。✅
- `run_command` **低风险命令白名单**：`cargo check/test`、`npm test`、`npm run build`、`git status/diff` 等在 Workspace Auto 下放行，无需 Full Access；含管道/串联/重定向的命令一律不算低风险（防绕过）。✅
- **Ask Every Time 真实弹窗**：后端 emit `approval-request` → 前端弹窗「允许一次/拒绝」→ oneshot 通道唤醒工具循环；拒绝则回灌模型让它换方案，被拒工具显示 ⊘ 卡片。✅
- 顺带修复 DSML 标记泄漏（详见 Plan11「DSML 兜底解析」参考章节）。✅

验收：

- Workspace Auto 下 Agent 可自行运行测试/构建/git status。✅
- Ask Every Time 下高风险动作弹窗确认，确认后执行、拒绝后跳过并告知模型。✅

### 0.0.13 - 错误自纠 + 项目认知

目标：让 Agent 失败后能恢复，开局就懂项目结构。

任务：

- **错误自纠引导**：工具/命令失败时，把结构化错误回灌并附纠错提示，引导模型读错误、换方案重试，而非直接放弃。
- **repo map / 工作区索引**：会话开局生成工作区结构摘要（Aider 风格），注入上下文，让模型无需逐层 list_dir 就知道项目骨架。

验收：

- 命令失败后 Agent 能基于错误信息调整并重试。
- Agent 开局即了解项目主要目录与文件，减少盲目探索。

### 0.0.14 - diff/patch 审阅底座

目标：让改动可审阅、为回滚打基础。

任务：

- `apply_patch`（单文件文本 patch）+ `show_diff`。
- 文件修改返回 before/after 摘要，前端可查看 diff。

验收：

- 用户修改源码时，Agent 先读文件、生成 patch、应用 patch。
- UI 能展示文件 diff。
